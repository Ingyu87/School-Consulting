# 고일초 찾아가는 학교 컨설팅 문서 생성 웹앱 PRD

## 1. 제품 목표

이 웹앱의 최종 목적은 발표용 슬라이드가 아니라, 심층면담 현장에서 학교와 함께 컨설팅 내용을 정리하고 최종 산출물인 `심층면담지`와 `운영계획서`를 바로 생성하는 것이다.

앱은 `서울고일초_사전 자가진단 분석.csv` 형식을 표준 파서 양식으로 삼아, 어느 학교의 사전 자가진단 CSV를 올려도 같은 방식으로 분석해야 한다. 고일초 자료와 일정 가안은 샘플/초기 프로젝트로 제공하되 모든 값은 현장에서 수정 가능해야 한다. 사용자는 진단 결과를 시각화해 학교에 설명하고, 면담 중 논의된 내용을 입력하며, 연수 모듈과 일정을 조정한 뒤, PDF 양식 스타일에 맞춘 `심층면담지.docx`와 `운영계획서.docx`를 다운로드할 수 있어야 한다.

## 2. 핵심 사용자

- 리더 코디네이터: 현장에서 학교 담당자와 면담하며 진단 결과를 설명하고 연수 구성을 확정한다.
- 참여 코디네이터: 모듈 구성, 일정, 논의사항, 학교 요구사항을 함께 입력한다.
- 학교 연수 담당자: 자가진단 결과와 추천 연수 구성을 확인하고 학교 여건에 맞게 결정한다.

## 3. 핵심 성공 기준

- 고일초 자가진단 결과가 한눈에 보이고, 낮은 영역과 운영상 확인할 사항이 명확히 드러난다.
- PDF 공식 안내 기준에 맞게 연수 구성 오류를 자동 검증한다.
- 일정표 CSV의 고일초 일정은 초기값으로 들어가지만 사용자가 자유롭게 수정할 수 있다.
- 사용자가 입력한 내용이 새로고침 후에도 유지된다.
- 최종적으로 `심층면담지.docx`와 `운영계획서.docx`를 다운로드할 수 있다.
- 문서 출력물은 폴더 내 `심층면담지.pdf`, `운영계획서.pdf`의 구조와 문체를 따른다.

## 5. 공식 운영 기준

앱에서 `안내` 또는 `검증`으로 제시하는 기준은 PDF에 있는 내용만 사용한다.

### 5.1 연수 구성 기준

- 총 5개 과정, 12차시 이상 의무 개설
- 필수과정 2개 + 선택과정 3개
- 필수과정 제외 최대 15차시 개설 가능
- 필수과정 포함 최대 17차시
- 모듈0 `우리 학교 알아보기`는 필수 1차시
- 모듈7 `우리 학교 변화 돌아보기`는 필수 1차시
- 모듈1, 모듈2, 모듈3 합계는 최대 5차시
- 교원 대상 연수는 최소 2차시 이상 구성
- 모든 수업은 오프라인 원칙
- 부득이한 경우 전체 연수 차시의 30%에 한해 모듈1~6 온라인 가능

### 5.2 식사·다과 기준

- 사업 기간 내 식사 1회, 간식 3회 제공
- 리더십 과정 진행 시 예산 내 식사 1회 추가 제공
- 학부모·학생 연수와 비대면(온라인) 연수에서는 다과·식사 미제공
- 식사 시간은 연수시간에 포함되지 않음

### 5.3 출석·이수 기준

- 연수 시작 전·후 출석 QR 태그 필요
- 연수 시작 전 참여자 정보 제출 QR 작성 필요
- 직무이수 희망 교사는 한국과학창의재단 종합·원격교육연수원 ID 필요
- 과정별 80% 이상 수강 시 직무이수 처리

## 8. 저장 정책

- 모든 입력은 `localStorage` 또는 `IndexedDB`에 자동 저장
- 상단에는 버튼이 아닌 상태 표시로 `자동저장됨` 표시
- 사용자가 원할 때 `백업 JSON` 다운로드 가능
- 백업 JSON을 다시 불러오는 기능 제공
- 새로고침, 브라우저 재시작 후에도 작업 복원

## 9. 기술 방향

### 9.1 HTML 하나로 가능한 범위

가능:
- 진단 시각화
- 입력 폼
- 자동 저장
- 연수 구성 검증
- JSON 백업
- 간단한 DOCX 생성

제한:
- 운영계획서/심층면담지와 거의 동일한 고품질 DOCX 표 레이아웃 생성은 단일 HTML만으로 구현 난도가 높음
- 템플릿 기반 DOCX 치환, 다중 페이지 표, 로고/푸터 재현은 별도 라이브러리나 빌드 구성이 안정적

### 9.2 권장 방향

Vercel 배포를 고려한다면 Vite 기반 정적 웹앱으로 시작하고, 문서 생성 품질이 더 필요해지는 시점에 Vercel Functions를 추가하는 구조로 재구성한다.

- React 또는 Vanilla TypeScript
- `docx` 라이브러리로 DOCX 생성
- `recharts` 또는 SVG 직접 구현으로 시각화
- `localforage` 또는 브라우저 IndexedDB로 저장
- Vercel 배포
- 선택 확장: `/api/generate-docx` Vercel Function으로 템플릿 기반 DOCX 생성

Vercel 배포 버전에서도 사용자의 학교 데이터와 면담 입력은 기본적으로 브라우저에 저장한다. 서버로 파일을 전송하는 기능은 사용자가 명시적으로 문서 생성을 요청할 때만 사용한다.

## 10. 비범위

- PDF 안내자료에 없는 운영 기준을 앱이 임의로 안내하지 않는다.
- CSV 일정 가안을 확정 일정처럼 표시하지 않는다.
- 운영기관 승인 여부를 앱이 판단하지 않는다.
- 사용자의 입력 없이 최종 운영계획서 문장을 과도하게 자동 확정하지 않는다.

## 11. 범용 진단 CSV 분석 요구사항

이 앱은 고일초 전용으로 끝나면 안 된다. 어느 초등학교 심층면담에 가더라도, 해당 학교의 사전 자가진단 CSV를 업로드하면 같은 방식으로 진단 결과를 분석하고 문서 작성 초기값을 만들 수 있어야 한다.

### 11.1 CSV 업로드

- 사용자는 `{학교명}_사전 자가진단 분석.csv` 파일을 직접 업로드할 수 있다.
- 파서 기준 양식은 `서울고일초_사전 자가진단 분석.csv`이다.
- 앱은 업로드된 CSV에서 학교명을 자동 추출한다.
- 학교명 추출이 실패하면 사용자가 직접 학교명을 입력한다.
- 고일초 데이터는 기본 샘플/초기 프로젝트로 제공한다.
- 새 CSV를 업로드하면 새 학교 프로젝트로 시작할지, 현재 프로젝트를 덮어쓸지 확인한다.

### 11.2 CSV 구조 대응

앱은 다음 구조를 자동 탐지한다.

- `STEP1. 사전 자가진단 종합 분석표`
- 과정별 평균 점수
- 과정별 문항명
- `STEP2. 학교 디지털 기반 교육 현황 알아보기`
- 문항별 응답 결과/비율
- 서술형 문항 위치
- `STEP3. 학교 디지털 기반 교육 혁신 현황 자가진단 데이터 분석`

CSV의 행/열 위치가 학교마다 조금 달라도, 텍스트 마커 기반으로 구간을 찾아야 한다.

### 11.3 자동 분석 결과

업로드 후 앱은 다음 결과를 자동 생성한다.

- 전체 평균
- 과정별 평균 점수
- 가장 낮은 3개 영역
- 가장 높은 3개 영역
- 3.8 미만 도약 단계 영역
- 3.8~4.6 만족 단계 영역
- 4.6 이상 추월 단계 영역
- 디지털 기기/무선망/유지보수 등 인프라 취약 문항
- 학생/학부모/교원/관리자 대상 연수 필요성
- 운영계획서에 넣을 수 있는 분석 초안
- 심층면담에서 확인할 질문 초안

### 11.4 분석 결과 편집

자동 분석 문구는 그대로 확정하지 않는다.

- 모든 분석 문구는 `분석 초안`으로 표시한다.
- 사용자는 문구를 직접 수정할 수 있다.
- 수정된 문구가 운영계획서 DOCX에 반영된다.
- 원본 자동 분석과 사용자 수정본을 구분해 저장한다.

## 12. 진단 시각화 요구사항

진단 분석 화면은 상세하되, 학교 담당자가 한눈에 이해할 수 있어야 한다. 기존 문서형 표 스타일만으로는 설명력이 부족하므로, 진단 분석 영역은 더 컬러풀하고 역동적인 대시보드로 구성한다.

### 12.1 시각화 원칙

- 문서 작성 화면은 PDF 양식 스타일을 따르되, 진단 분석 화면은 컬러풀한 데이터 대시보드로 구성한다.
- 색상은 정보 구분을 위해 사용하되, PDF 공식 문서 톤과 충돌하지 않도록 남색 기반에 보조 색상을 제한적으로 쓴다.
- 그래프는 발표 중 바로 설명할 수 있을 만큼 크고 명확해야 한다.
- 모든 그래프에는 해석 문장을 함께 제공한다.

### 12.2 필수 그래프

- 과정별 평균 점수 가로 막대그래프
- 과정별 단계 구분 도넛 또는 분포 바
- 낮은 영역 TOP 3 카드
- 높은 영역 TOP 3 카드
- 인프라 문항 응답 분포 stacked bar chart
- 기기/네트워크/유지보수 위험도 요약 카드
- 추천 모듈 연결 그래프: 낮은 진단 영역 → 추천 연수 모듈
- 12차시 구성과 진단 결과 연결 타임라인

### 12.3 애니메이션

애니메이션은 깔끔하고 업무용 도구에 맞게 사용한다.

- CSV 업로드 후 분석 카드가 순차적으로 나타난다.
- 평균 점수 숫자는 count-up으로 표시한다.
- 막대그래프는 왼쪽에서 오른쪽으로 부드럽게 채워진다.
- 위험도/강점 카드는 0.25~0.4초 범위의 fade/slide 애니메이션을 사용한다.
- 탭 이동 시 과한 전환 효과는 금지한다.
- 문서 작성 폼에서는 애니메이션보다 안정적인 입력 경험을 우선한다.

### 12.4 컬러 규칙

- 강점/높은 점수: Blue/Teal 계열
- 보완 필요/낮은 점수: Amber/Coral 계열
- 위험/기준 위반: Red 계열
- 중립/보통: Gray/Slate 계열
- 모듈별 색상은 0~7번 과정에 고정 배정하여 다른 학교 CSV에서도 일관되게 사용한다.

## 13. 범용 학교 프로젝트 요구사항

### 13.1 프로젝트 생성

- `새 학교 시작` 버튼 제공
- CSV 업로드 후 학교명 기준으로 프로젝트 생성
- 예: `서울고일초`, `서울송파초`, `서울가동초`

### 13.2 프로젝트 저장

- 프로젝트별로 자동 저장
- 브라우저에 여러 학교 프로젝트를 보관할 수 있어야 한다.
- 프로젝트 목록에서 학교를 전환할 수 있어야 한다.
- 백업 JSON에는 학교명, 업로드 CSV 분석 결과, 면담 입력, 연수계획 입력을 모두 포함한다.

### 13.3 문서 파일명

다운로드 파일명은 업로드된 학교명에 따라 자동 생성한다.

- `{학교명}_심층면담지_{YYYYMMDD}.docx`
- `{학교명}_운영계획서_{YYYYMMDD}.docx`

고일초라는 이름이 코드에 고정되면 안 된다.
